# Open decisions

The architecture above is committed. These remain to be decided.

1. Surrogate-net build vs adopt, for the D4 surrogate formulas.
2. PDE-mesh **adjoint scheme** (discrete- vs continuous-adjoint of the
   finite-volume operator) for the macro continuum tier. The mesh **format** is no
   longer open — it is committed as a `DeviceMesh` finite-volume `Universe` with
   fields as fibers (`multiscale-state §7`); only the adjoint binding
   remains, reusing the Stage-4→Stage-5 AD seam.
3. ~~The `γ̂` open questions of `gamma-hat §4` (ε-equality,
   materialization policy, long-trajectory drift / rank-refresh, rank-dependent
   applicability of the LowRank slot).~~ **Closed 2026-07-21.** They were four
   faces of one problem — exact content-addressed identity meeting approximate
   numerics — and resolve under `[representation-substrate]` §4.1/§20.4.2 plus
   the scorer-only decision. Each disposition is recorded at
   `[gamma-hat]` §4: ε-equality → the rewrite-admission rule
   (`[compose-time-pipeline]` §3); materialization → the Stage-4 adjoint-tape
   schedule (`[compose-time-pipeline]` §4), which was a scheduling question all
   along; drift → exported to the integrating consumer via the manifest's
   encoding validity domain (below); rank applicability → a Stage-4 predicate.
   Kept struck through rather than deleted so the list a reader sees matches the
   history.
4. Layer-1.75 minimum spec sufficient for a V2 contributor to implement
   self-consistent GW / DMFT.
5. **A semiconductor-interface applicability predicate.** Row 116
   (`interface-trap-density`) needs one and the classifier vocabulary has nothing
   broad enough: `has-metal-semiconductor-interface` is too narrow, because traps
   sit at semiconductor/dielectric interfaces too. Opened 2026-07-22 when row 116
   was found gated on `is-noncentrosymmetric` — a point-group predicate with no
   bearing on dangling-bond density, which had been refusing trap density for
   every centrosymmetric host including diamond
   (`[applicability-classifiers]` §1). Row 116 carries an always-true stub until
   the predicate exists; naming the gap rather than inventing a classifier is
   `[traps]` §61.
6. **Which language fills each of the four polyglot roles.** The *shape* is
   closed (below): a compiler host owning Stages 1–4 and the substrate, emitting
   source for a separate runtime host, plus an offline group-theory engine and an
   offline proof assistant. The picks are not. `[deriv-language-study]` recorded
   Haskell / Julia / GAP / Lean 4 as its leading candidates and — more usefully —
   *which requirement each was chosen to satisfy*; that requirement list is what a
   later comparison should carry forward. Nothing in the build order depends on the
   pick, which is why it can stay open. Every mention of a language elsewhere in
   this corpus is a candidate to compare against, never a mandate.

**Verifier-soundness gaps (named, not silently absent).** Four were named; the
identity/ε rule (`[representation-substrate]` §4.2) closed three of them on
2026-07-21. They stay listed with their resolutions rather than deleted, because
an absent check reads exactly like a passing one, and a *silently* closed one
reads like it was never there:

1. **No post-registration adjoint-drift monitoring.** *(Still open — and the ε
   rule does not close it, which is worth being explicit about.)* The vJp-vs-JvP
   gate runs once, at registration, against the **formula's** adjoint
   (`[residual-machinery]` §5). Stage-4 synthesizes the **composition's**
   adjoint later, per composition, over a graph Stage 3 has already rewritten.
   Those are different objects, and nothing re-validates the second. The
   rewrite-admission rule (item 2) makes a rewrite's *value* discrepancy visible;
   it says nothing about whether the adjoint synthesized downstream of it is still
   the adjoint that was gated.
2. ~~No Stage-3 rewrite-soundness contract.~~ **Closed 2026-07-21.** The
   rewrite-admission rule is now normative at `[compose-time-pipeline]` §3: exact
   over ℝ, float side conditions discharged by an e-class interval / not-equals
   analysis, and a registered fidelity generator. The mechanism is the one that
   made Herbie's rule set sound (Zhang et al., PLDI 2023).
3. ~~No declared validity radius for frozen Layer-1.25 dressings.~~ **Closed
   2026-07-21** — a frozen dressing is an approximation, so under §20.4.2 it owes
   an estimator like any other, and the estimator *is* the radius. To first order
   the dropped term is `‖Δx‖ · ‖∂(dressing)/∂x‖_ref`: the sensitivity coefficient
   is measured once at the reference state at compile time (the dressing is
   already computed there), and the runtime factor is a norm on the state. Both
   cheap; `OneShotCert` (`[residual-machinery]` §7) gains the coefficient
   field, and the product enters `combineTol` as the dressing-staleness term that
   already exists but had no number in it.
   **Still open, and separable:** G₀W₀ cost is not scope-tagged by cell size. At
   MVP scale a T2 budget (≤10 s) is plausible; for V1 defect supercells it is
   hours, so the tier table is violated silently rather than refused. That is a
   *cost* claim, not an accuracy one, so the ε rule does not reach it.
4. ~~Gradient error under truncated inner solves is unbudgeted.~~ **Closed
   2026-07-21.** `τ_cond` bounds the Jacobian's conditioning at registration;
   `τ_trunc` (`[cert-obligations]` §1.2) is the a-posteriori estimate of the
   product `‖J⁻¹‖·‖r_stop‖`, emitted per instance by the fidelity generator
   (`[residual-machinery]` §4.1) and composed into `combineTol`. The
   estimate is a byproduct — the stopping residual is already computed.

The conditioning guard on implicit-function adjoints, which belonged to this
list, closed on 2026-07-21: `τ_cond` (`[cert-obligations]` §1.2), gated at
registration for every `D3` row.

**Unregistered composition formulas.** `[typed-compositions]` invokes eighteen
formula names that are not registry rows, so its closed-vocabulary claim does not
yet hold in full. The gap is declared in that page's `unregistered-formulas`
frontmatter and machine-checked: a *declared* name is tracked, an *undeclared*
one fails the seam sweep. Two tranches, needing different work — about half are
definitional (the expression is inline, or the name plus its inputs determine it,
or `[traps]` already pins the form) and are transcription plus tag assignment;
the rest are modelling choices needing literature (four hardness models, the
HTST and jump-diffusivity prefactor conventions, turnover frequency,
formation-energy-from-references). Deliberately not registered on thin
provenance: a row must be defensible against a citation, and the registry's
whole discipline is that unsourced values are refused. Four invocations
*used to* carry inline mathematics, which `[named-formulas]` forbids outright;
they now carry their declared names, and the expressions those names stand for
are recorded on that page — which is what makes registering the four definitional
rows transcription rather than research.

## Closed decisions

- **Multiscale state (slow + macro tiers) & the device scale-bridge** =
  the state is stratified into three tiers (`multiscale-state`): the micro
  7-tuple (unchanged), a **slow / configurational tier** (defect populations,
  H content, oxidation/carbide fronts) evolving by Arrhenius generation–annihilation
  kinetics — **aging is core `/physics`** state the PINO predicts and `/physics`
  scores — and a **macro continuum tier** (homogenized `T_L(r), φ(r), n(r), p(r),
  j(r)` on a `DeviceMesh`) bridged to the micro tier by an explicit homogenization
  map. The emergence axiom is refined to *same-timescale/scale* coarse-graining
  (resolving the `unified-state`↔`born-oppenheimer-levels` tension); the full distribution stays emergent
  by moment closure (no DAE); two EOM-family residual categories
  (`EOM/DefectPopulation`, `EOM/Continuum`) are added. No new computational method is
  introduced (the slow tier reuses `kinetic-evolution`). Residue: the PDE-mesh
  adjoint scheme (open item 2 above).
- **Implementation-language *shape*** (the *picks* are open item 5 above) = a
  **polyglot of domain-specific DSLs**
  joined at the pipeline's Stage-4→Stage-5 codegen seam, not a single language
  (`[deriv-language-study]`). **The shape is settled; which language fills each
  slot is not.** What the study established is structural and language-neutral:
  a compiler host that owns Stages 1–4 and the `representation-substrate` substrate, *emitting
  source* for a separate runtime host that JIT-compiles it once per composition
  (so nothing crosses the seam per sample but flat arrays); an offline
  group-theory engine for the Stage-2/2.5 character tables and Reynolds
  projectors; and an offline proof assistant for the §20.4 injectivity and
  algebraic-law obligations. Those four roles are the decision.
  **The candidates below are the study's leading picks, recorded so a later
  comparison starts from evidence rather than from scratch — not a commitment.**
  Each slot must be re-argued against its alternatives before code is written,
  and the requirement each candidate was chosen to satisfy is the durable part.
  - **Haskell** is the leading compiler-host candidate. It hosts Stages 1–4 + the `representation-substrate` substrate (the symbolic-IR
    compiler): GADTs / `DataKinds` / type families type the op-indexed Merkle DAG
    and the `SymbolicTensorOps` operad at compile time; `hegg` provides the
    Stage-3 e-graph; `GHC.Generics` derives the §20.4 canonical serializer; our
    own AD drives Stage-4 adjoint synthesis. (OCaml is the documented fallback
    host.)
  - **Julia** is the leading Stage-5 runtime candidate: Stage 4 emits source,
    JIT-compiled once per composition, so the hot loop is native and there is no
    per-sample FFI; the runtime host would own the optional GPU codegen. The
    requirement is a host with fast JIT of generated numeric source and a
    workable GPU story — not Julia specifically.
  - **GAP** is the leading offline group-theory candidate for generating and
    validating the Stage-2/2.5 character tables and Reynolds projectors
    (`|G|≤192`), baked into the compiler as content-addressed tables. Any
    engine that produces auditable tables for `|G|≤192` satisfies the
    requirement. (Note `GAP` is overloaded three ways in this corpus — see
    `[traps]` §59.)
  - **Lean 4** is the leading offline proof-assistant candidate for the §20.4
    pre-hash canonical-encoding injectivity proof and the `EvidenceOps` /
    `GroupOps` / ROBDD algebraic laws. The requirement is a proof assistant that
    can state and check those obligations.

  Hardware: a **CPU-dominant compiler**; GPU is a Stage-5-only optional accelerator
  chosen per-composition (the symmetry quotient yields small irrep blocks — a GPU
  anti-pattern — so the diamond MVP often runs fastest on CPU). The polyglot split
  is a net win **conditional on** a Stage-4→Stage-5 differential golden test; the
  single-host fallback is Julia-only or Haskell-only. Rust (the single-language
  winner) is excluded by preference; Python+JAX is disqualified — its tracer owns
  differentiation, which conflicts with bring-your-own adjoint synthesis.
- **Integrator interface (`dynamics`)** = the **per-tier evolver hand-off — a
  causalized tangent kernel plus a steppable-form manifest — not an integrator**
  (decided 2026-07-16 on the scorer↔stepper duality research,
  `live/specs/2026-07-16-evolver-duality-research.md`; verdict: survives with
  restrictions). `dynamics(tier)` exposes: the pure tangent map
  `(state_tier, env, adiabatic-params) → tangent_tier`; separately addressable
  generator sub-entries (`E`, `S`, `δE/δx`, `δS/δx`, and the `L·`/`M·`
  contraction blocks — required by degeneracy-respecting / discrete-gradient
  integrators); the algebraic subsystem with per-step solve plans and an
  index-≤1 witness; per-block generator tags with preservation grades; the
  obligation map (`ResidualKey → conserve | bound | monotone`); the tier
  cadence/coupling contract; declared per-step cost; the **encoding validity
  domain** (below); and the sibling fingerprint + certificate reference.

  **Encoding validity domain (added 2026-07-21).** `/physics` is scorer-only
  (`[gamma-hat]` §2) — nothing accumulates in here — so a consumer
  that *integrates* a tier inherits the representation-health problem along with
  the tangent map. The manifest therefore declares the encoding each block was
  compiled against and the conditions under which that encoding stops being a
  fair approximation: the `CompressionPlan` slot, its rank, and its truncation
  target. Exporting the problem is legitimate; exporting it silently is not.

  This composes with the obligation map already listed above. That map's
  `conserve | bound | monotone` vocabulary is exactly the vocabulary the robust
  dynamical-low-rank literature states its guarantees in, so a consumer can match
  an integrator to the obligations term-for-term. The **rank-adaptive
  basis-update & Galerkin integrator** (Ceruti, Kusch & Lubich, *BIT* 62, 2022)
  is the named family: up to a declared truncation tolerance it preserves the
  norm when the equation does (→ `conserve`, e.g. `Tr γ̂`), the energy for
  Hamiltonian systems (→ `conserve`, the `L` block), and monotone decrease of the
  functional in gradient flows (→ `monotone`, the `M` block). Its error bounds
  are **independent of small singular values** (Ceruti & Lubich, *BIT* 62(1)
  23–44, 2022, which carries that robustness over to the BUG family; first proved
  for projector-splitting by Kieri, Lubich & Walach, *SIAM J. Numer. Anal.* 54,
  2016) — the property no standard integrator has, and the one
  that makes a low-rank `γ̂` safe to step at all. Naming it is a declaration, not
  an implementation obligation on `/physics`. `/physics` guarantees exactness against
  the scorer sibling, tag totality, structural witnesses, and refusal
  accounting; the consumer — `/informed-operator`, an integration harness in
  `/interface`, or a user program — owns scheme choice, step-size control, and
  the loop. Score-not-solve is preserved: the hand-off is a per-call readout of
  the instantaneous lawful tendency, never a trajectory ([product]).
  Realized as a flag-gated Stage-4 sibling emission sharing the scorer's
  content-addressed RHS forests (one Stage-3 extraction, two lowerings).
  Residue: the full lowering spec (manifest record, refusal enum, the
  scorer–evolver exactness obligation) lands as its own named wave — slow tier
  first — and no time-evolution product verb is claimed until it does.
- **ReferenceCache backend** = `SqliteReferenceCache` (`cert-obligations §2`).
- **Coverage-mask format** = `RoaringCoverageMask` (`pino-bridge §2.1`).
- **Curriculum schedule** = `0.10 / 0.60 / 0.90` cumulative fractions, last
  `0.10` cooldown (`residual-definitions §4.1`); overridable by
  `/informed-operator`.
- **Active-learning loop placement** = `/interface`
  (`purpose-and-scope` "What `/physics` is not").
- **Applicability decidability** = first-order decidable on typeclass tags;
  enforced at registration (`named-formulas`,
  `build-sequence` Phase 7).
- **Coupling structure** = generic `CouplingChannel` record with three
  parameter axes (`pieces`, `target ∈ {Scalar, AntisymmForm, PSDSymmForm}`,
  `order × derivative`) plus a Stage-2.5 invariant generator that constructs
  the explicit terms from the crystal's symmetry group
  (`coupling-structure`). The nine named cross-regime strings of
  `generic-dynamics` collapse to a handful of channel declarations; the
  per-coupling terms are generated, not registered.
- **Coupling-channel coverage policy** = a bounded `CoverageBound`
  (global cap `(order ≤ 4, Gradient(1))` + a per-mechanism inner table)
  plus the runtime rule "active channels are those whose `applicability`
  holds and whose invariant basis is non-empty," **not** a hand-curated
  registry (`coupling-structure §9`). Each channel carries a
  `mechanism_range` tag whose derived `polynomial_sufficient` flag decides
  whether the symmetry-generated basis is the whole coupling; long-range
  channels carry a typed `KernelExt` for the non-polynomial part
  (§19.10–§19.11). The composition's theory frame is a `TheoryContext` on
  `CouplingSpec` (XC functional / pseudopotentials / many-body level /
  relativistic treatment), and `PSDSymmForm` channels carry documented PSD
  assumptions rather than a runtime SDP search (§19.11–§19.12). MVP default
  theory context: `PBE / PseudoDojo-v0.4.1-NC / KohnSham / ScalarRelativistic`.
- **Representation substrate** = one substrate contract (typed indexed
  universes, content-addressed Merkle DAGs, sparse-set backends,
  persistent stage-visible maps) plus a small parametric op-signature
  family (`PredicateOps`, `SymbolicTensorOps`, `EvidenceOps`,
  `GroupOps`) and specialized algebraic fibers per cluster
  (`representation-substrate`). Every `/physics` representational object
  is a fiber over this substrate; identity, canonical serialization,
  and attachment are unified, while each cluster keeps the backend that
  wins on its hot path.
- **PhysicsGraph identity** = closure of output `Address[GraphNode]`
  set under children-pointers; edges are the children addresses inside
  each node payload, not separately identified.
- **Sidecar / evidence relationship** = sidecars carry `EvidenceId`
  attachments via the `EvidenceBearing` wrapper; materialized evidence
  lives in a persistent `EvidenceDAG`. Sidecars remain ephemeral or
  cache-eligible per domain; evidence persists across composition
  boundaries.
- **No solver-call hot paths.** Every hot operation in the
  `/physics` substrate is `O(log n)` or better with documented
  constant factors (`representation-substrate §5`); no equivalence
  or subsumption op requires SAT or a runtime rewrite engine.
