# The named-formula registry

The canonical, machine-readable list is
`physics/library/formulas/registry-manifest.csv` — 132 substantive rows plus 2
markers for relations enforced architecturally and therefore not residualized
(force = −∇energy; equivariance). `formula-registry.md` is the narrative index.
Every algebraic combination invokes a named formula with typed inputs and an
explicit output type; no inline math.

Each formula record carries:

```
record FormulaRecord {
  name               : Symbol                  -- behavior-named (e.g. defect-formation-energy)
  signature          : (Inputs) → Output       -- typed, with units
  bundle             : {BundleId}              -- one or more of B1..B11, or the
                                               --   L1 primitive tag (linear-response
                                               --   primitives Z*/ε∞/χ∞/α_M, rows 91–94,
                                               --   feed multiple bundles)
  cost-tier          : T0 | T1 | T2 | T3
  diff-tag           : D0 | DN | D1 | D2 | D3 | D4
  path               : cheap | faithful          -- anchor class, not a runtime
                                                 --   path selector; see below
  source             : provenance pointer (research file / literature DOI)
  depends-on         : {Symbol}                -- upstream formulas / primitives
  applicability      : (Crystal, Environment) → Bool
  adjoint-validated  : Passed | Failed(witness) | NotApplicable | Relaxed(rationale)
}
```

**Cost tiers:** T0 closed-form (≤10 µs) · T1 small linear algebra / 1D quadrature
(≤10 ms) · T2 Brillouin-zone / mesh integral (≤10 s) · T3 self-consistent loop or
PDE solve (≤10 min).

**Differentiability tags.** The tag answers exactly one question: *how does a
consumer obtain a gradient through this row?*

- **`D0` — pure read.** The output is a stored or passed-through value; the
  adjoint is the identity. Registers without a synthesized adjoint. Exactly one
  row qualifies: `reference-phase-energy-cache`, keyed on a phase id alone. A
  row that also takes continuous arguments is not a pure read, however
  cache-backed its implementation — see the note on `D0` misuse below.
- **`DN` — no useful derivative.** Integer, categorical, boolean or set-valued
  output (topology invariants, discrete classifications). Not differentiable and
  not relaxable in place: a consumer needing a gradient must route around it, or
  use a relaxation registered as its own row. **`DN` is the strongest claim in
  this vocabulary** — it asserts that no relaxation exists, not merely that none
  is written yet — so it is the tag most likely to be wrong. Before assigning it,
  check whether the output has a real-valued component (see the mixed-output rule
  below) and whether this corpus already prescribes a relaxation elsewhere.
- **`D1` — smooth; gradient available directly.** Analytic closed form, or a
  smooth composition the AD engine handles. No registration gate.
- **`D2` — adjoint required, validated at registration.** vJp-vs-JvP agreement
  on sampled points within `τ_adj` (`impl-07-residual-factory §7.5`). The gate
  checks the *synthesized* adjoint, not a hand-written backward.
- **`D3` — implicit-function adjoint.** The output is a converged fixed point;
  the gradient is one linear solve against the transposed fixed-point Jacobian,
  independent of forward iteration count. Conditioning is set by that Jacobian,
  and near-singular Jacobians are the failure mode. **The iteration-count
  independence is the test**: if the adjoint's cost scales with how many forward
  steps were taken — a backward-in-time sweep through a transient — the row is
  `D2`, not `D3`.

  **`D3` is a refinement of `D2`, not an alternative to it.** It names *how* the
  adjoint is synthesized; it does not exempt the row from validation. Every `D3`
  row runs the same registration gate as `D2`, **and one more**: a conditioning
  check on the fixed-point Jacobian at the sampled points, refusing registration
  when the reciprocal condition number falls below `τ_cond`. Without that second
  check the tag would be strictly weaker than `D2` — which is backwards, since
  the ill-conditioned fixed point is the one failure mode canon names for it, and
  the gradient it produces is *large and wrong* rather than absent. The rows this
  bites are not exotic: `fermi-level-charge-neutral` and
  `self-consistent-charge-balance` solve charge neutrality, whose Jacobian
  `∂F/∂E_F` is flattest exactly in a wide-gap intrinsic semiconductor — this
  corpus's subject — and `SCPH-self-consistent-phonons` is invoked precisely
  where soft modes make its Jacobian near-singular.
- **`D4` — relaxed.** Genuinely non-smooth (argmin, hull, sort, discrete
  metric). Ships a declared smooth relaxation whose bias is a model-form error
  entering `combineTol`, approved at registration with an obligation-9 validity
  domain. **The relaxation is named in the row's `source` cell**; a `D4` row
  without one is un-gateable and fails the registry-build gate.

**Mixed-output rows.** Several rows return a real quantity *and* a discrete
label derived from it — `radius-ratio-coordination-class` (`ratio + class`),
`elastic-stability-criteria` (`bool-vec + slack`), `structure-uniqueness-CSP`
(`bool + d`). **The diff-tag describes the continuous component.** The discrete
label is a downstream classification carrying no gradient of its own, and a
consumer needing a gradient differentiates the continuous component. A discrete
label never drags a row to `DN` — otherwise the same construction would be
tagged three different ways, which is how these three rows previously read
`D1`, `D1` and `DX`. Where the *map* from continuous to discrete is itself
load-bearing (a threshold, a min-over-set), that non-smoothness is what makes
the row `D4`, and the relaxation covers exactly that step.

**`DN` splits the old `D0`**, which conflated *"a pure read, whose adjoint is
the identity"* with *"an integer output, which has no derivative at all"*.
Those are opposite answers to the question the tag exists to answer, and the
conflation was load-bearing: eleven smooth analytic rows sat at `D0` and so
registered with **no adjoint at all**, since `impl-11-verification` exempts
`D0`. Among them was the alloy-lattice interpolation — linear in composition —
which `product` promises is directly optimizable through the baked gradients.
Those eleven are now `D1`. A twelfth, `chemical-potential-ref-table`, survived
the first pass because "ref-cache lookup" reads like a pure read; it takes `T`
and `P`, so tagging it `D0` zeroed `∂μ_i/∂T` and left row 69's Maxwell
cross-derivative residual vacuously satisfied. That is the shape of the `D0`
error to watch for: an implementation detail (a cache) mistaken for a
mathematical one (an identity adjoint).

**The tag is spelled `DN`, not `DX`.** `DX` collides with the DX center — the
deep-donor configuration this corpus discusses in AlGaN and AlN
(`[deriv-defects]` Part B) — so a search for the tag returned the physics and a
search for the physics returned the tag. Diff tags are written in backticks in
prose for the same reason; the wurtzite deformation potentials `D1`–`D5` in
`[accuracy-ledger]` are the other live collision, and backticks are what
separate them.

The corrected physics is canonical in the registry and in §6 below: optical
absorption uses `(2ω/c)·Im(√ε)`; the operator-spectrum-area sum rule carries the
`2/π` prefactor; the acoustic sum rule sums over all lattice translations
(`Σ_J Σ_R Φ_{IαJβ}(R) = 0`); the magnetic relaxation term is the
orientation-preserving form `S × (S × H_eff)`; the harmonic transition-rate
normalization consumes products-over-modes (scalars), not spectra.

**`path` — the anchor class.** `cheap` (117 rows) versus `faithful` (15). This
is **not** a runtime path selector: under the always-cheap pipeline
(`[arch-07-pipeline]`) every registered formula lands on the single residual
surface, and nothing chooses between two paths at runtime. What the field records
is what a row's value is *anchored against*: a `cheap` row stands on its own
closed form, while a `faithful` row is one whose value is only trusted against a
reference-grade computation — a DFT/DFPT/NEGF/MC evaluation or a measured
battery entry.

That makes it the axis obligation-6 pairs run along. A **consistency pair** is a
`cheap` row and a `faithful` row computing the same observable with no agreement
theorem between them, only a bounded model gap (`[impl-09-cross-cutting]`); the
QHA + Slack/Callaway κ against the iterative-LBTE κ is the canonical example, and
treating it as an equivalence pair would score the legitimate gap as a bug
(`[traps]` §29). The `Cheap vs faithful` column of `[accuracy-ledger]` is the
per-observable statement of the same distinction.

The field was briefly declared *retired* on the grounds that the always-cheap
reframe collapsed the two runtime paths. It did — but the anchor-class meaning is
what the corpus actually uses the column for, in four live places, and one of them
is the rule that defines consistency pairs. Retiring a field that four pages
depend on is how a load-bearing distinction becomes folklore.

**Applicability-decidability invariant.** Every `applicability` predicate is
first-order decidable in `(Crystal, Environment)` — finite case analysis on
typeclass tags (lattice type, site decoration, environment-field presence),
not on numeric thresholds or solver outputs. Non-decidable classifiers are
forbidden at registration; see the registry-build gate
(`impl-10-build-sequence` Phase 7).
