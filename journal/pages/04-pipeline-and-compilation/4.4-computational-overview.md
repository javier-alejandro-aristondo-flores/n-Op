# `/physics` ŌĆö Computational Architecture Reference

> A **computational-lens** reference for `/physics`: the system described as a *program* ŌĆö
> data structures, algorithms, complexity, numerical behavior, data layouts, order of
> operations. **Physics meaning is deliberately suppressed** (when `E`, `S`, `L`, `M`, `╬│╠é`
> appear, they denote a functional, an operator, a matrix ŌĆö not their physical content).
> This is a hand-written companion to the atomic spec (`the canon chapters*`,
> `the canon chapters*`), **not** part of the lint-enforced atomic tree. Every claim cites
> its canonical `arch-xx` / `impl-xx` source so it can be verified rather than re-derived.
> Depth target: an engineer can derive low-level structures, layouts, solver choices, and an
> order of operations from this document. It does **not** prescribe a language or write code.

---

## 1. What kind of program this is

`/physics` is **a compiler that emits a numerical kernel**, organized around **one
content-addressed data substrate**.

- **Compose-time** (the compiler): builds a typed symbolic IR (a hash-consed DAG), rewrites it
  through four passes, and *lowers it to a runtime kernel*. Runs **once per problem instance**
  ŌĆö a `(PeriodicityStructure, SiteDecoration, Environment)` tuple ŌĆö in secondsŌĆōminutes
  (`arch-07-pipeline ┬¦7.6`). Branchy, allocation-heavy, pointer-chasing; latency- and
  correctness-bound, not throughput-bound.
- **Runtime** (the emitted kernel): a straight-line numeric function `(state, env) ŌåÆ numbers`,
  run **millions of times** at ┬ĄsŌĆōms each. No symbols, no structural branching, no solver
  invoked from scratch.

Polyglot realization (`arch-18-open-decisions`): the compile-time host and the runtime host
are **separate**, and the compiler *emits source* for the runtime host, which JIT-compiles it
once per composition ŌĆö so the hot loop is native and only flat arrays cross the seam. Two more
roles sit offline: a finite-group engine for character tables, and a proof assistant for
substrate-invariant proofs. **The four-role split is settled; the languages filling the roles
are still open** ŌĆö `[deriv-language-study]` records the leading candidates (Haskell / Julia /
GAP / Lean 4) and the requirement each was picked to satisfy. Language choice is orthogonal to
everything below; this document is language-neutral.

The central design lever (`arch-07-pipeline`): **all expensive structure discovery ŌĆö symmetry
reduction, sparsity pattern, low-rank/compression plan, adjoint factorization ŌĆö is done once
at compile time; the runtime only *applies* precomputed structured operators.** This is what
lets every runtime hot path be `O(log n)` or better and call no solver (`arch-20 ┬¦20.5`).

## 2. The memory model ŌĆö the representation substrate (`arch-20`)

Every load-bearing object is a *fiber* over five primitives. This is the storage and identity
contract for the whole system.

### 2.1 Primitives

| Primitive | Computational object | Core operation costs |
|---|---|---|
| `Address[D]` | 256-bit content hash (SHA-256), domain-separated | construct `O(payload)`; equality `O(1)` (32-byte compare Ōēł two SIMD loads) |
| `Universe[U]` | interned vocabulary ŌåÆ dense integer ordinals (`u32` closed / `u64` open) | member equality `O(1)` (one word); ordinal doubles as a dense array index |
| `SparseSet[U]` | subset of a universe, density-chosen backend | membership `O(log n)`ŌĆō`O(1)`; bulk set-ops backend-specific |
| `PersistentMap[K,V]` | immutable HAMT, branching factor 32 | lookup/insert `O(logŌéāŌéé n)`, path-copy with structural sharing |
| `MerkleDAG[S,L]` | hash-consed DAG, parameterized by op-signature `S`, leaves `L` | root equality `O(1)`; structural diff `O(changed frontier)` |

`Node[S,L] = Leaf(L) | Op(op : S.Op, attrs : S.Attrs[op], children : S.Arities[op])`. The
op-signature `S` makes attribute types and arities a function of the op tag.

### 2.2 Content addressing and the canonical serializer (`arch-20 ┬¦20.4`) ŌĆö the #1 correctness surface

`Address[D] = SHA-256(domain_separator(D) ŌĆ¢ schema_version(D) ŌĆ¢ canonical_node_bytes)`.
**Identity is the hash of the canonical bytes**, so the serializer's *injectivity* is the
single highest-consequence invariant in the system (two distinct values hashing equal breaks
all caching, dedup, and equality). The 11 rules:

1. 16-byte domain separator first (prevents cross-domain collision even on identical bytes).
2. `u32` schema version (bumped on incompatible change; old addresses never silently reinterpreted).
3. Records: fields **lexicographically sorted by name**, each `name_bytes ŌĆ¢ length-prefixed value`.
4. Sequences: `u64` length prefix, elements in declared order.
5. Sets: elements canonicalized, **sorted by address bytes**, length-prefixed (order-independence).
6. Maps: `(key_bytes, value_bytes)` **sorted by key bytes**, length-prefixed.
7. Sum types: `u32` discriminant ordinal ŌĆ¢ length-prefixed payload.
8. Continuous factors: U(1) sectors ŌåÆ signed integer lattice weights + basis-id; SU(2) ŌåÆ doubled spin labels (`2j : Nat`) + ClebschŌĆōGordan table-version address.
9. Group action laws: normalized homomorphism/twist generators, relation rows sorted by `(domain, codomain, action_label)`.
10. `Optional`: `0x00` (None) vs `0x01 ŌĆ¢ value` (Some) ŌĆö never confusable with empty-but-present.
11. Scalars: `Nat`/`Int` fixed-width **big-endian**; `Float` IEEE-754 binary64 big-endian with **two normalizations applied first ŌĆö every NaN ŌåÆ one canonical quiet-NaN; `ŌłÆ0.0 ŌåÆ +0.0`** ŌĆö so numerically-equal floats always share an address.

**Numerical note:** rule 11's float normalization is what makes content-addressing safe for
computed values; without it, `ŌłÆ0.0`/`+0.0` or distinct NaN bit-patterns would fork identity.
Length-prefixing (not delimiters) prevents concatenation ambiguity. Hashing is the only
`O(payload)` operation on the identity path; everything downstream is `O(1)` address compare.

### 2.3 Backend-selection decision procedures (apply these directly)

**`SparseSet` backend ladder** (`arch-20 ┬¦20.5`; a property of the *universe's* density estimate, fixed at registration):
- `n Ōēż 8` ŌåÆ **sorted tuple** (free-monoid backend): no allocation, linear scan, cache-resident; `O(n)` membership with tiny constant.
- dense, `n Ōēż 256`, small closed universe ŌåÆ **bitset**: word-parallel union/intersection.
- sparse over a large universe ŌåÆ **Roaring**: `O(log n)` container probe, SIMD bulk ops, compresses runs.
- persistent + diffable ŌåÆ **HAMT (branching 32)** or **Merkle trie**: `O(logŌéāŌéé n)`, path-copy.

**`PersistentMap`**: HAMT depth `Ōēż ŌīłlogŌéāŌéé |keys|Ōīē` (Ōēż ~6 for Ōēż 10Ōü╣ keys); insert path-copies one root-to-leaf path, leaving the rest shared (enables cheap stage-to-stage sidecar versioning).

**`MerkleDAG`**: node store = `Map<Address, Node>`; equality is one root-address compare; `diff` descends only subtrees whose addresses differ (skips equal subtrees in one compare) ŌåÆ diff cost is proportional to the changed frontier, not total size.

### 2.4 Op-signatures and the per-cluster storage table (`arch-20 ┬¦20.2ŌĆō┬¦20.3`)

Four op-signatures specialize the Merkle DAG:
- **`PredicateOps`** ŌĆö reduced ordered binary decision diagrams (ROBDD) over versioned atoms (applicability, `arch-13`).
- **`SymbolicTensorOps`** ŌĆö a colored-operad / free symmetric monoidal category over target shapes `{Scalar, AntisymmForm, PSDSymmForm}` with tensor product, contraction, derivative, group action, projection, binding (the tensor-algebra IR for invariants and `FormulaApply`).
- **`EvidenceOps`** ŌĆö attestation, semilattice-meet aggregation, reference linkage, trajectory chunk (cert, `arch-12`).
- **`GroupOps`** ŌĆö multiply, invert, restrict, antipode/TR-twist, character, projector (finite-group algebra).

|---|---|---|---|
| C1 | 10 closed/open vocabularies (`StateComponent`, `NodeKind`, `BundleId`, `AxisLabel`, `CategoryTag`, ŌĆ”) | `Universe[T]` + dense ordinals | ordinal compare |
| C2 | registered generators (`FormulaRecord`, `ResidualGenerator`, `CouplingChannel`) | registry-universe element, kind-indexed dispatch | `Address[Element]` |
| C3 | sidecars (Stage 1/2/2.5/4) | `PersistentMap[TypedKey, EvidenceBearing[V]]`, HAMT-32, stage-visible | root address |
| C4 | evidence (the EvidenceDAG) | `MerkleDAG[EvidenceOps, EvidencePayload]`, persistent | root address |

| C5 | all content addresses (node id, `ResidualKey`, generator-hash, cache keys) | SHA-256 domain-separated (┬¦2.2) | 256-bit digest |

## 3. The data operated on ŌĆö state and `╬│╠é` (`arch-04`, `arch-15`, `arch-08`, `arch-21`)

**Three-tier state (`arch-21-multiscale-state`).** The state is stratified by
timescale/scale into **micro** (the 7-tuple below, unchanged), **slow** (defect
populations / aging ŌĆö first-class state evolving by Arrhenius generationŌĆōannihilation
kinetics, which the PINO predicts and `/physics` scores), and **macro** (homogenized
device-mesh fields `T_L(r), Žå(r), n(r), p(r), j(r)`). The full carrier distribution
stays *emergent* by moment closure (`T_e`, `j` closed-form), so no distribution-as-state
DAE arises. The slow tier is an adiabatic parameter of the micro tier; the macro tier is
homogenized from it (micro `╬║(T), Žā(T), ╬▒(E)` ŌåÆ device-PDE coefficients). The 7-tuple
below is the micro tier; the slow and macro tiers, the homogenization map, and the two
extra EOM-family residuals (`EOM/DefectPopulation`, `EOM/Continuum`) are in `arch-21`.

### 3.1 The 7-tuple `x = (h, R_I, P_I, ╬Ā_h, Z_I, ╬│╠é, A)` ŌĆö computational types

| Component | Container | Layout / notes |
|---|---|---|
| `h`, `╬Ā_h` | 3├Ś3 real matrices | 9 doubles (72 B) each; trivially cache-resident |
| `R_I`, `P_I` | `N├Ś3` real arrays | row-major by atom ŌåÆ per-atom locality; `N` = #atoms |
| `Z_I` | integer vector, length `N` | immutable species labels; dense ordinals |
| `A` | field on a grid | dense array, FFT-friendly layout (reciprocal-space ops) |
| `╬│╠é` | encoded operator (below) | the one feasibility-critical structure |

Slots are a closed `StateComponent` vocabulary (`Universe[StateComponent]`); the whole tuple is
content-addressed by ┬¦2.2. Downstream code indexes slots by dense ordinal, not symbol.

### 3.2 `╬│╠é` ŌĆö the load-bearing data structure (`arch-15`)

Logically an `N├ŚN` Hermitian operator; physically a **single logical object with a 5├Ś4
encoding grid** `(Basis Ōłł {Real, Reciprocal, Wannier, NaturalOrbital, SymmetryAdapted}) ├Ś
(Form Ōłł {Dense, Sparse, BlockDiag, LowRank})`. Stage 4 selects exactly one slot per
density-matrix node.

**Selection decision procedure** (by `(PeriodicityStructure, SiteDecoration)`, `arch-15 ┬¦15.1`):
- periodic bulk ŌåÆ `(Reciprocal, BlockDiag)` (MVP default)
- defect / surface / amorphous ŌåÆ `(Real, Sparse)`
- interface layers / dangling bonds ŌåÆ `(Wannier, Sparse)`
- low-rank substrate ŌåÆ `(NaturalOrbital, LowRank)`
- output of symmetry-adapted construction ŌåÆ `(SymmetryAdapted, BlockDiag)`

**Data layout of the MVP slot `(Reciprocal, BlockDiag)`** (`mvp-02-gamma-budget`): one block per
k-point; **each block stored as orbitals `N_PW ├Ś N_b` (low-rank in the band index), never the
dense `N_PW ├Ś N_PW` matrix.** Complex doubles (16 B). k-blocks are mutually independent ŌåÆ
embarrassingly parallel and independently addressable. Budget:
`N_PW ├Ś N_b ├Ś 16 B ├Ś N_k`. Concrete MVP: `N_PW Ōēł 1000` (Ōēł400 eV plane-wave cutoff),
`N_b Ōēł 40`, `N_k Ōēł 29` irreducible (8├Ś8├Ś8 mesh) ŌćÆ **Ōēł 18 MB**. The dense form of the same data
would be `N_PW┬▓ ├Ś 16 ├Ś N_k Ōēł 460 MB` ŌĆö **the slot choice is the feasibility boundary, not an
optimization.** Warm-start seed: a tight-binding `~18├Ś18` Hamiltonian per k (kilobytes) feeds
the SCF inner loop. Beyond MVP, supercells grow `N_PW` linearly; orbital storage stays linear
in `N_atoms ├Ś N_b` *provided the encoding is never densified*.

**Read/write asymmetry** (`arch-15 ┬¦15.2`) ŌĆö optimize for read:
- **Read path (dominant, every trajectory step):** lazy materialization via destructor methods ŌĆö
  apply operator (`matmat` against the `N_PW ├Ś N_b` factors), extract density (outer products),
  trace (inner products), eigendecomposition. Costs are set by `N_b` (the rank), not `N_PW┬▓`.
- **Write path (rare ŌĆö construction, self-consistency; **not** time-stepping ŌĆö `/physics` is scorer-only, `arch-15 ┬¦15.2`):** staged through a planner;
  self-consistency's gradient handled by the Stage-4 implicit-diff adjoint (┬¦5), not by unrolling.

**The four `╬│╠é` data-structure questions (`arch-15 ┬¦15.4`) ŌĆö resolved, not open.** They were one
problem seen four ways: exact content-addressed identity meeting approximate numerics. The
resolution is `arch-20 ┬¦20.4.1`/`┬¦20.4.2` ŌĆö **identity stays exact; ╬Ą is estimated beside it** ŌĆö
plus the scorer-only decision. Summarised here; `arch-15 ┬¦15.4` is canonical:
1. **╬Ą-equality:** identity is *not* bisimulation-up-to-╬Ą and must not be ŌĆö `Ōēł_╬Ą` is not transitive,
   so it yields no quotient and nothing to hash. Rewrites are admitted when exact over ŌäØ with float
   side conditions discharged by an e-class interval / not-equals analysis (┬¦4 below, `arch-07 ┬¦7.3`).
2. **Materialization policy:** misfiled. Forcing vs deferring changes *cost*, not *value* ŌĆö no error
   term exists. It is rematerialization-vs-storage (`revolve`: optimal on a chain, NP-hard on a DAG),
   and belongs to the Stage-4 schedule over the adjoint tape.
3. **Long-trajectory drift:** not this library's problem ŌĆö nothing accumulates in a scorer. Exported
   to the integrating consumer via the steppable-form manifest, with the robust dynamical-low-rank
   family (projector-splitting / BUG; error bounds independent of small singular values) named as the
   structure-preserving answer.
4. **Rank-dependent applicability:** a compile-time predicate on `(PeriodicityStructure,
   SiteDecoration)` decided at Stage 4, not a runtime check ŌĆö the runtime cost was the whole
   objection, and there is no longer a runtime decision to pay for.

### 3.3 Level layering L1ŌĆōL4 (`arch-08`) ŌĆö a computational dependency order

Dependencies flow strictly **upward** `L4 ŌåÆ L3 ŌåÆ L2 ŌåÆ L1`; each level introduces irreducible
state. The level of a node is **derived from its transitive inputs, not stored**; Stage 1
constructs the graph in level order.

| Level | Operates on | Operations (computational) |
|---|---|---|
| L1 | `╬│╠é`, `A` at fixed `(R,h)` | apply operator, eigendecomposition, trace, density extraction (single-particle linear algebra on the `╬│╠é` encoding) |
| L2 | `(R,P,h,╬Ā_h)`, `Z` fixed | **nested minimization** `E_BO(R,h) = min_{╬│╠é} E[╬│╠é;R,h]` (L1 inner loop), derivative-based forces, MD timestepping |
| L3 | spectra from L1/L2 | spectral reductions / statistical aggregation (quadrature over eigenvalues) |
| L4 | distributions over L1ŌĆōL3 | kinetic/transport: sparse linear solves, transition-matrix methods, sampling |

**Order-of-operations consequence:** an L2 force evaluation contains a *converged L1 inner
solve*; the adjoint must thread the implicit-diff chain through both. The
L3Ōåönon-equilibrium cycle closes by a same-pass fixed point (`impl-11 ┬¦11.2`: Ōēż 5 iterations in
the worked example).

### 3.4 Dressing tiers (`arch-08 ┬¦8.1`) ŌĆö an implementation-scoped layering, not a runtime hierarchy

- **Layer 1** ŌĆö bare substrate.
- **Layer 1.25** ŌĆö one-shot, closed-form corrections (single non-iterated solves); evidence = `OneShotCert`. **MVP runs entirely here.**
- **Layer 1.75** ŌĆö iterative fixed-point corrections (each needs a bespoke Stage-4 lowering); evidence = `IterativeResult`. **V2-deferred**; V1 ships type/cert scaffolding with loud `not-implemented` stubs (`impl-07 ┬¦7.7`).

## 4. The IR ŌĆö the PhysicsGraph (`arch-06`)

A **hash-consed, content-addressed DAG**. Recommended low-level representation: an **arena /
index DAG** ŌĆö nodes in a flat array, `NodeId` = integer handle (not a pointer) ŌĆö which gives
compact storage, cache locality on traversal, trivial serialization, and a natural hash-cons
table keyed by `Address`.

```
Node = ( id   : Address[GraphNode]          -- hash-cons identity
       , type : Layer0Type                  -- the 4 typeclasses (arch-10)
       , kind : NodeKind , role : OutputRole )

NodeKind   = Input(InputKind)            -- InputKind = StateSlot(StateComponent) | EnvScalar(EnvField)
           | FormulaApply(formula : NamedFormula, args : [NodeId])     -- 132 formulas
           | MethodInvoke(method  : NamedMethod,  args : [NodeId])     -- 12 methods
OutputRole = Internal | Observable(bundle : BundleId) | ResidualLeaf(ResidualKey)
```

- **Edges are the `args` lists** inside apply/invoke nodes ŌĆö there is no separate edge table;
  the graph *is* the multiset of output-root addresses closed under children-pointers.
- **Identity / hash-consing:** identical subgraphs collapse to one address (`O(1)` amortized
  via the address table). Two graphs are equal iff their output-root multisets match.
- **Per-stage sidecars** (`arch-06 ┬¦6.4`): ephemeral `Map<NodeId, T>` produced by one stage and
  visible to later stages per the poset, **not** hash-consed, erased before runtime. Stage-visibility poset
  `1 < 2 < 2.5 < 3 < 4 < 5`. Backed by HAMT-32 (C3).
  - `Stage1Sidecar.applicability : Map<NodeId, Predicate>` (discarded after pruning)
  - `Stage1Sidecar.coupling-channels : List<CouplingChannel>` (consumed by Stage 2.5)
  - `Stage2Sidecar.symmetry : Map<NodeId, IrrepBlock>` (consumed by Stage 4)
  - `Stage2_5Sidecar.invariants : Map<CouplingChannel, GeneratorOutput>` (consumed by Stage 3)
  - `Stage4Sidecar.compression : Map<NodeId, CompressionPlan>`, `.adjoint : Map<FixedPointNodeId, AdjointSolver>` (erased after codegen)

**`CompressionPlan` family (`arch-06 ┬¦6.4`) and its Stage-4 decision procedure:**
`Dense | Sparse(pattern) | LowRank(rank) | HODLR(params) | TT(ranks)`. Choose per operator node by:
- small or dense operator ŌåÆ **Dense**;
- known/inferred sparsity pattern (from Stage-3 sparsity inference) ŌåÆ **Sparse**;
- numerical rank `r Ōē¬ n` (estimate via rank-revealing QR or randomized SVD at Stage 4) ŌåÆ **LowRank(r)**;
- hierarchically low-rank off-diagonal blocks ŌåÆ **HODLR**;
- high-dimensional tensor operator (e.g. a collision kernel) ŌåÆ **TT(ranks)**, cores built once via sequential TT-cross.

## 5. The core algorithm ŌĆö the 4+1 pipeline (`arch-07`)

Each compose-time pass rewrites the IR; the fifth is the emitted numeric loop. The table gives
inputŌåÆoutput, the algorithm/math class, and cost; details follow.

| Stage | In ŌåÆ Out | Algorithm class | Cost (compile-time unless noted) |
|---|---|---|---|
| 1 Symbolic lift | request + descriptors ŌåÆ pruned IR | macro expansion + Boolean decision diagrams | seconds; ROBDD eval `O(path len)` |
| 2 Symmetry quotient | IR ŌåÆ block-structured IR | finite-group representation theory (rewrites only) | seconds; group sums `O(\|G\|)`, `\|G\| Ōēż 192` |
| 2.5 Invariant synthesis | coupling decls ŌåÆ tensor-algebra DAGs | Reynolds projection over a finite group | seconds; Ōēż 12 M ops worst case, cached |
| 3 Algebraic simplification | IR ŌåÆ shared sparse IR | term rewriting / e-graph (union-find + e-matching) | the algorithmically hardest pass |
| 4 Lower + adjoint + codegen | IR ŌåÆ kernel | numerical-LA planning + implicit-diff AD + codegen | secondsŌĆōminutes |
| 5 Runtime apply | (state, env) ŌåÆ numbers | the 12 numeric kernels | ┬ĄsŌĆōms ├Ś millions (runtime) |

**Stage 1 ŌĆö lift + prune.** Order: instantiate each requested template (a graph-construction
macro) into a subgraph of `Input` / `FormulaApply` / `MethodInvoke` nodes in **L1ŌåÆL4 level
order**; evaluate each node's applicability predicate (ROBDD, `O(decision-path)`, three-pointer
fetch per node); **delete** any subgraph whose applicability is false for this `(Crystal,
Environment)`. Output is a pruned graph; the applicability sidecar is discarded.

**Stage 2 ŌĆö symmetry quotient (rewrites, no numerics here).** Two transformations that *reduce
the work Stage 5 will do*:
- **Schur block-diagonalization:** an operator commuting with the group action is rewritten into
  its per-irrep blocks (block dim `╬Ż_╬╗ d_╬╗ m_╬╗`). This turns a single `O(n┬│)` dense eigensolve
  into `╬Ż_╬╗ O((d_╬╗ m_╬╗)┬│) Ōē¬ O(n┬│)`. **Numerical note:** the blocks are *small* (cubic point
  groups give `d_╬╗ Ōłł {1,2,3}`, up to 4 under spinŌĆōorbit), which favors a cache-resident dense
  solve per block and is a poor fit for wide-SIMD/GPU batching unless many blocks are batched.
- **IBZ orbit collapse:** orbitŌĆōstabilizer reduces a full-zone k-mesh to the irreducible wedge
  with integer orbit weights (cubic: up to 48├Ś). Requires character tables (GAP, offline) and a
  SchreierŌĆōSims product table (`O(1)` avg multiply).

**Stage 2.5 ŌĆö invariant synthesis (`arch-19 ┬¦19.3`).** For each active `CouplingChannel`, in order:
1. **Spinor-parity pre-prune** (`O(#pieces)`): odd total spinor parity cannot form the target shape ŌåÆ empty basis, computed before any character work.
2. **Character-inner-product emptiness test:** `Ō¤©Žć_T, Žć_trivialŌ¤® = (1/|G|) ╬Ż_g Žć_T(g)` (`O(|G|)` traces, Ōēż ~200 ops, **never materializing `Žü(g)`**) predicts whether the invariant basis is non-empty *before* building it.
3. If non-empty, apply the **Reynolds (trivial-irrep) projector** `P = (1/|G|) ╬Ż_g Žü(g)` to the
   target tensor `T` (`O(|G|┬Ędim(T)┬▓)`, Ōēż ~12 M ops worst case). Result cached on
   `Address[CrystalSymmetryGroup] ├Ś Address[CouplingChannel]`.
   Bounds enforced: `|G| Ōēż 192`, `dim(T) Ōēż 250`, Ōēż 12 M ops. Output: one `GeneratorOutput`
   per channel (`arch-19 ┬¦19.3` ŌĆö the `InvariantTerm` basis as
   `MerkleDAG[SymbolicTensorOps, ŌĆ”]` plus the `polynomial_sufficient` echo and any
   `kernel_extension`), lowered in Stage 3 into `FormulaApply` nodes on the
   `E_coupling` / `L_assembly` / `M_assembly` aggregators (`arch-05`).

**Stage 3 ŌĆö algebraic simplification.** Hash-consing (intern identical subexpressions, `O(1)`
amortized), cross-formula common-subexpression elimination (the 132 formulas share
intermediates), tearing / alias elimination, sparsity-pattern inference. Implemented as
equality-saturation over an e-graph (union-find of equivalence classes + e-matching). Symbolic;
no numeric cost. **This is the hardest pass to build** and the one whose performance is
open-ended; granularity is preserved (each `ResidualLeaf` keeps its content-addressed key).

**Stage 4 ŌĆö lowering + adjoint synthesis + codegen.**
- **Compression-plan selection** per operator node (the ┬¦4 decision procedure; may run a
  randomized-SVD range-finder to estimate numerical rank).
- **Implicit-differentiation adjoint synthesis** for fixed-point / implicit nodes (SCF
  minimization, BTE-RTA): the gradient of a converged fixed point is **one linear solve against
  the (transposed) fixed-point Jacobian**, independent of the forward iteration count ŌĆö versus
  differentiating through the unrolled loop. Conditioning of that solve is set by the fixed-point
  map's Jacobian; near-singular Jacobians (slow self-consistency) are the failure mode.
- **Registration-time adjoint gate (`impl-07 ┬¦7.5`):** every `D2` and `D3` generator runs a
  forward-vs-reverse consistency check (vJp vs JvP) on `N Ōēł 64` sampled points, and `D3`
  additionally a conditioning check on the fixed-point Jacobian (`Žä_cond`); if the max
  relative error exceeds **`Žä_adj = 1e-4`** the build **fails loud**, forcing an honest adjoint
  or an explicit retag to `D4` with a named relaxation and recorded rationale (`D3` is a
  structural claim about fixed points, not a fallback tier ŌĆö `impl-07 ┬¦7.5`).
- **Codegen:** lower the whole graph to one kernel with one entry (the `Input` slots) and typed
  exits (residual map, gradient map, observable map, cert evidence). Sidecars erased.

**Stage 5 ŌĆö runtime apply.** Apply the compiled kernel to a dense state vector. The 12 methods
(┬¦6) are the numeric kernels invoked here. Forward pass cost = kernel-DAG evaluation; the
optional adjoint pass is reverse-mode by structural projection, `O(residual-vector size)`.

## 8. Outputs and the consumer boundary (`arch-11`, `arch-16`)

The emitted kernel's single entry point:

```
Validate(state, env, request : all|{ResidualKey}|{ObservableRef}, gradient : Skip|Compute)
-- the pino-bridge surface (arch-16 ┬¦16.1); it wraps the raw kernel primitive
-- `evaluate : (State, Environment) ŌåÆ (residuals, gradient, ŌĆ”)` of arch-11 ┬¦11.4

```

- **`ResidualKey = (producer, axes)`**, `producer = Formula(NamedFormula) | Method(NamedMethod)`;
  content-addressed (C5). Output is **granular by construction** ŌĆö one scalar per
  `(producer, axis-tuple)`, no pre-aggregation; the consumer chooses the reduction. The 19
  `CategoryTag`s (9 EOM-family ŌĆö 7 micro per-component + the slow/macro
  `EOM/DefectPopulation`/`EOM/Continuum` siblings, 3 GENERIC-structural
  `Degeneracy`/`Conservation`/`Positivity`, 5 `Algebraic/*` identities, 2 `Static/*`
  constraints) are a **facet** in a parallel `Map<ResidualKey, ContributionFacets>`, **never
  part of identity and never a loss weight** (`impl-09 ┬¦9.3`).
- **Gradients:** one reverse-mode pass over the selected subgraph, projected per key ŌåÆ
  `O(output size)`. Granularity adds only that leaves are individually addressable; upstream
  sharing is already deduplicated by Stage-3 hash-consing.
- **`request` / `gradient`** prune the evaluated subgraph and toggle the adjoint.
- **`Import`** injects external ground truth `(named-target : ObservableRef, value, Žā, provenance, coverage-mask) ŌåÆ
  GroundTruthBridgeGenerator`; it inserts a pinned `Input` + a cert-only `ResidualLeaf`, is **not
  differentiated through**, and feeds cert obligation 4.
- **`RoaringCoverageMask`**: a serialized Roaring bitmap over `enumerate(product(axes))` (lexicographic
  flat-index), marking which axis-tuples a datum constrains. Sparse-from-start; `O(1)` membership,
  fast intersection/union/cardinality for cert set-ops; persisted in `SqliteReferenceCache`.
- **The consumer reads by content-hash key (`ResidualKey` / `ObservableRef`), never by node id.**
