# The named-formula registry

The canonical, machine-readable list is
`physics/library/formulas/registry-manifest.csv` — 132 substantive rows plus 2
markers for relations enforced architecturally and therefore not residualized
(force = −∇energy; equivariance). `formula-registry.md` is the narrative index.
Every algebraic combination invokes a named formula with typed inputs and an
explicit output type; no inline math.

Each formula record carries:

```
record FormulaRecord {
  name               : Symbol                  -- behavior-named (e.g. defect-formation-energy)
  signature          : (Inputs) → Output       -- typed, with units
  bundle             : {BundleId}              -- one or more of B1..B11, or the
                                               --   L1 primitive tag (linear-response
                                               --   primitives Z*/ε∞/χ∞/α_M, rows 91–94,
                                               --   feed multiple bundles)
  cost-tier          : T0 | T1 | T2 | T3
  diff-tag           : D0 | DX | D1 | D2 | D3 | D4
  source             : provenance pointer (research file / literature DOI)
  depends-on         : {Symbol}                -- upstream formulas / primitives
  applicability      : (Crystal, Environment) → Bool
  adjoint-validated  : Passed | Failed(witness) | NotApplicable | Relaxed(rationale)
}
```

**Cost tiers:** T0 closed-form (≤10 µs) · T1 small linear algebra / 1D quadrature
(≤10 ms) · T2 Brillouin-zone / mesh integral (≤10 s) · T3 self-consistent loop or
PDE solve (≤10 min).

**Differentiability tags.** The tag answers exactly one question: *how does a
consumer obtain a gradient through this row?*

- **`D0` — pure read.** The output is a stored or passed-through value; the
  adjoint is the identity. Registers without a synthesized adjoint.
- **`DX` — no useful derivative.** Integer, categorical, boolean or set-valued
  output (topology invariants, discrete classifications). Not differentiable and
  not relaxable in place: a consumer needing a gradient must route around it, or
  use a relaxation registered as its own row.
- **`D1` — smooth; gradient available directly.** Analytic closed form, or a
  smooth composition the AD engine handles. No registration gate.
- **`D2` — adjoint required, validated at registration.** vJp-vs-JvP agreement
  on sampled points within `τ_adj` (`impl-07-residual-factory §7.5`). The gate
  checks the *synthesized* adjoint, not a hand-written backward.
- **`D3` — implicit-function adjoint.** The output is a converged fixed point;
  the gradient is one linear solve against the transposed fixed-point Jacobian,
  independent of forward iteration count. Conditioning is set by that Jacobian,
  and near-singular Jacobians are the failure mode.
- **`D4` — relaxed.** Genuinely non-smooth (argmin, hull, sort, discrete
  metric). Ships a declared smooth relaxation whose bias is a model-form error
  entering `combineTol`, approved at registration with an obligation-9 validity
  domain.

**`DX` is new and splits the old `D0`**, which conflated *"a pure read, whose
adjoint is the identity"* with *"an integer output, which has no derivative at
all"*. Those are opposite answers to the question the tag exists to answer, and
the conflation was load-bearing: eleven smooth analytic rows sat at `D0` and so
registered with **no adjoint at all**, since `impl-11-verification` exempts
`D0`. Among them was the alloy-lattice interpolation — linear in composition —
which `product` promises is directly optimizable through the baked gradients.
Those eleven are now `D1`.

The corrected physics is canonical in the registry and in §6 below: optical
absorption uses `(2ω/c)·Im(√ε)`; the operator-spectrum-area sum rule carries the
`2/π` prefactor; the acoustic sum rule sums over all lattice translations
(`Σ_J Σ_R Φ_{IαJβ}(R) = 0`); the magnetic relaxation term is the
orientation-preserving form `S × (S × H_eff)`; the harmonic transition-rate
normalization consumes products-over-modes (scalars), not spectra.

**Applicability-decidability invariant.** Every `applicability` predicate is
first-order decidable in `(Crystal, Environment)` — finite case analysis on
typeclass tags (lattice type, site decoration, environment-field presence),
not on numeric thresholds or solver outputs. Non-decidable classifiers are
forbidden at registration; see the registry-build gate
(`impl-10-build-sequence` Phase 7).
