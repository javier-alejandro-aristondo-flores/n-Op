# Cross-cutting design rules

These rules cut across formulas, methods, residuals, cert, and the
pino-bridge surface. They are not architectural decisions ŌĆö those live in
`arch-*`. They are reusable patterns that show up in more than one place
in the implementation.

## 1 Method equivalence as a residual, not a path-selector

Several observables admit multiple formulas that should agree on their
shared domain (conductivity via BTE-RTA vs Kubo; thermal conductivity via
QHA+Callaway vs DFPT+3-phonon-BTE; effective mass from `k┬Ęp` vs DFT
band-curvature). The discipline under the always-cheap pipeline
(`compose-time-pipeline`) and the granularity rule (`residual-definitions`)
is:

- All applicable formulas for an observable are instantiated as
  `FormulaApply` nodes (`physics-graph ┬¦2`); hash-consing
  collapses shared subgraphs.
- Equivalence is enforced via the `Algebraic/MethodEquivalence`
  residual category (`residual-definitions ┬¦1`), one `ResidualLeaf`
  per agreeing pair.
- Cert obligation 6 (`cert-obligations`) consumes the
  `Algebraic/MethodEquivalence` leaf; if `|fŌéü ŌłÆ fŌéé| > tolerance` it
  trips with both values as witnesses. The tolerance kind depends on the
  pair's sub-kind (`residual-definitions ┬¦1` cat-15): an **equivalence pair**
  (BTE-Žā ŌēĪ Kubo-Žā, sharing an agreement theorem) trips on any disagreement,
  while a **consistency pair** (QHA+Callaway ╬║ vs iterative-LBTE ╬║ ŌĆö *no* agreement
  theorem, only a bounded model gap) trips only on excess beyond the declared
  `Žä_method`. Treating the Callaway-vs-BTE ╬║ pair as an equivalence would wrongly
  score the legitimate model gap as a bug; it is a consistency pair.
- The `Observable` output role (`physics-graph ┬¦3`)
  designates which compose-time-selected formula is the *exposed*
  value to downstream consumers; selection is by `ContributionFacets`
  precedence (declared dressing tier, then registration order). The
  unselected formulas still contribute their residual leaves.

The architecture never averages observables and never silently selects
between formulas at runtime; both are exposed in the graph and the
disagreement is a typed residual.

## 2 Same-type ŌåÆ shared interpretation; type-change ŌåÆ explicit stage

Elements that are *parallel interpretations of one signature* ŌĆö the 19
`CategoryTag`s (`residual-definitions ┬¦1`), the dressing tiers within L1
(`born-oppenheimer-levels ┬¦1`), the 10 cert obligations
(`cert-obligations`), source/dressing tags on `ContributionFacets`,
applicability guards (`applicability-classifiers`) ŌĆö share one interface
with multiple handlers.

Elements that *compose into a pipeline with a type change between
stages* ŌĆö the ╬│╠é encoding pipeline (`gamma-hat`), the 4 BO
levels (`born-oppenheimer-levels`), the compose-time pipeline (stages 1ŌĆō4 + the 2.5 sub-stage)
(`compose-time-pipeline`), the synthesis ŌåÆ property ŌåÆ PINO layering ŌĆö stay
explicit multi-stage structures.

Rule of thumb: same type ŌćÆ one interface, many handlers; type changes
between stages ŌćÆ keep the stages.

## 3 Provenance tags are not weighting axes

`ContributionFacets` (`residual-definitions ┬¦2`) attaches `(category,
bundle, dressing)` to every `ResidualLeaf` as a sidecar ŌĆö purely
queryable provenance, never part of `ResidualKey` identity and never
the basis for a per-residual loss weight. Loss weighting lives in
`/informed-operator`'s curriculum schedule (`residual-definitions ┬¦4.1`),
keyed by `CategoryTag` participation gates only. A facet field exists
to answer "which residuals belong to bundle B?", not "what is the
weight of residual r?".

## 4 Couplings are declared channels; terms are generated

Cross-regime physics is not a hand-rolled list. The library author
declares `CouplingChannel` records (`coupling-structure`),
each specifying `{pieces, target, order, derivative, applicability}`.
The Stage-2.5 invariant synthesizer takes the crystal's symmetry
group and the active channels and *generates* the explicit
`InvariantTerm`s ŌĆö symmetry-respecting tensors that become
`FormulaApply` nodes in the graph. Adding a new physical regime
(piezoelectricity, magnon-phonon coupling, ŌĆ”) is a channel
declaration, not a code edit. Channels register through the same
factory pattern as residual generators (`residual-machinery ┬¦3`):
content-addressed by parameter tuple, applicability validated as
first-order decidable.
