# Corpus conventions

Rules for the book under `journal/`. Pages are the source of truth; the
apparatus (`contents.md`, `index.md`) is generated from their frontmatter.

The **authority order** and the corpus-wide rules immediately below
govern the whole base.

## Authority order

When two sources disagree, the higher-ranked one wins:

1. **The `canonical-for` owner.** The page whose frontmatter claims the
   topic is the single source of truth for it. Topics are globally
   unique and machine-checked, so this resolves most conflicts outright.
2. **`physics/library/formulas/registry-manifest.csv`** — canonical for
   formula rows, names, signatures, and cost/differentiability tags.
3. **`physics/library/cert/reference-data/*.csv`** — canonical for
   seeded coefficient values.
4. **Canon pages** (`authority: canon`). Peers: a disagreement between
   two canon pages is resolved by which one owns the topic, not by
   chapter order.
5. **Appendix derivations** (`authority: supporting`, chapter 11).
   Record how a result was derived, not what is currently true.
6. **`journal/live/`** — work products still executing.
7. **`README.md`.**

- **Seed from the ledger and the reference CSVs, never from an appendix
  derivation.** The appendix is the lowest-authority stratum and carries
  superseded values behind per-file changelogs; reseeding from it
  silently reverts a landed fix.
- **Generated outputs are never edited or audited.** `contents.md` and
  `index.md` are emitted by `journal/tools/apparatus.py`, which also
  restamps `content-hash`; `--check` gates their freshness. `glossary.md`
  is **hand-maintained**, not generated — `seams.py` only checks it for
  duplicate entries.
- **Frozen material is `journal/live/audits/` only.** An audit is a dated
  record of what was found on a day; rewriting it destroys the record, so
  internal staleness there is by-design and exempt. **`journal/live/specs/`
  is not frozen.** A spec is a work product *still executing* — the next
  agent researches against it — so a stale claim in a spec misdirects work
  rather than preserving history. Specs track current truth and carry a
  `## Changelog` like any other stratum.
- **Strata are rewritten to current truth**, with the superseded value
  preserved in a per-file `## Changelog` section rather than left
  standing in the body.

## Naming is addressing

Registry formula names are hash-consed into content addresses. A
post-landing rename is a substrate-wide rekey, not an edit — every
cached kernel, certificate, and address keyed on the old name breaks.
**Fix names before they land, never after.**

Until they land, every rename leaves its old name behind in prose.
`physics/library/formulas/retired-names.csv` maps each retired name to
what it landed as, and `seams.py` fails on any retired name that a page
mentions without resolving it in the same paragraph. That is what keeps
"naming is addressing" true of the *documentation* and not only of the
substrate.

**Eponyms are renamed when they misdescribe, not merely because they are
eponyms.** `grueneisen-mode` and `matthiessen-mobility` name exactly what
they compute and stay. `vibronic-coupling-factor` did not — vibronic
coupling also names the electron-phonon matrix element `g_qν`, so the row
shadowed a different quantity; it is now
`lattice-relaxation-phonon-quanta`. Nor did
`field-activated-ionization-rate`, which called a per-length coefficient a
rate while shadowing two other rows that are also field-activated
ionization; it is now `impact-ionization-coefficient`. The test is
whether a reader could bind the name to the wrong object, not whether a
person's name appears in it; the person goes in the `source` cell as an
`a.k.a.`, which is where a literature search starts anyway.

## File layout

The corpus is a **book** under `journal/`.

- One canonical topic per page. Pages live in one folder per chapter,
  `pages/NN-chapter-name/<chapter>.<n>-<slug>.md`. The folder is
  zero-padded so it sorts; the `<chapter>.<n>` prefix is a **display
  tag** — regenerated freely, never cited. A page's folder must match
  its `chapter` frontmatter; the checker enforces this.
- `pages/11-appendix-derivations/` holds derivations. They carry
  `authority: supporting` and rank below canon in the authority order
  above.
- `journal/live/` holds work products still executing. When one
  finishes, its conclusions are promoted into canon and the artifact
  itself collapses into `[timeline]`.
- `journal/tools/` holds the checkers. At the journal root: `contents.md`
  and `index.md` are **generated**; `glossary.md` and `instructions.md`
  are **hand-maintained**. `instructions.md` is the entry point an agent
  reads first, so it is the file whose staleness costs the most —
  `seams.py` covers it.

## Frontmatter (mandatory)

Every atomic file starts with a YAML frontmatter block:

```yaml
---
id: <permanent address — never changes>
title: <human-readable title>
chapter: <1..11>
chapter-name: <slug>
tag: <chapter>.<n>          # display only; regenerated
status: draft | review | stable
authority: canon | supporting
content-hash: <12 hex>      # tool-maintained
canonical-for:
  - <topic this page is the single source of truth for>
depends-on:
  - <id of upstream page>
referenced-by:
  - <id of downstream page>
research-sources: []
---
```

Field rules:

- **`id`** — the page's **permanent address**. Globally unique, and
  deliberately *not* derived from the filename or the tag, so a page can
  be renamed, retagged or moved between chapters without breaking a
  single citation. Anchors in prose use the bare id, written
  `[arch-06-physics-graph]`. Historical ids keep their `<tree>-NN-` shape;
  that is inherited, not a rule for new pages.
- **`canonical-for`** — every concept appearing in this list must
  appear in *exactly one* file's `canonical-for`. The lint enforces
  this; that's how we prevent drift.
- **`depends-on` / `referenced-by`** — symmetric. If `A.depends-on
  contains B`, then `B.referenced-by must contain A`. Empty lists are
  written `[]`. Lint enforces.
- **`status`** — `draft` until reviewed; `review` while a rewrite is
  in flight; `stable` once accepted.
- **`revision`** — incremented on every substantive content change.
  Metadata-only edits (populating `depends-on` / `referenced-by` /
  `research-sources`) do **not** bump `revision` — revisions track
  content drift, not graph housekeeping.
- **`depends-on` edge criterion** — `A depends-on B` iff A's body
  references B's id, **or** A uses a term whose `canonical-for` owner
  is B. Nothing else creates an edge. The first half is **machine-derived**:
  `apparatus.py` extracts every `[id]` from the citable body and fails if
  the edge is missing. The second half is authorial — a topic string like
  "conventions" is an ordinary English word, so matching it mechanically
  produces noise, and a checker that cries wolf gets ignored.
- **Citable body** — what can create an edge, for the derivation above:
  the prose, minus fenced code blocks, minus the `## Changelog` section,
  minus blockquote banner lines. A changelog entry recording that a value
  moved, or a banner saying a page is superseded, mentions another page
  without depending on it. `[timeline]` and `[traps]` are exempt in both
  directions: they are indexes over the corpus, and edges to them from all
  58 pages would carry no information.
- **The page graph is cyclic, and that is correct.** `arch-04-state` and
  `arch-05-generic` each explain the other. There is no acyclicity check
  and there will not be one. **Nothing may compute a transitive closure
  over `depends-on` and read it as a build order or a layering** — the
  corpus is one large strongly-connected component. `depends-on` answers
  "what should I read alongside this", not "what must be built first".
- **`research-sources`** — paths under `physics/research/` only;
  other docs go under `depends-on`.

## Anchor and cross-reference style

- Cross-references use the `id`, written either inline as
  `[arch-06-physics-graph]` or with section coordinate as
  `arch-07-pipeline §7.4`. Section coordinates must resolve against a
  real heading in the target file (lint check 11).
- Section numbers inside a page are inherited from its pre-book
  ancestry (e.g. `arch-11-residuals` uses §11.x) so that section
  coordinates in existing citations keep resolving. They are *not*
  related to the chapter number. New pages use whatever numbering reads
  cleanly, or dated/slug headings — see `[timeline]`, which is cited by
  heading text rather than by number.
- Never link by file path. Links go through `id`s so file moves don't
  rot references.
- **Id-less targets** — files outside `pages/`, which have no `id`:
  `journal/live/`, `journal/tools/`, `journal/instructions.md`, and the
  CSVs under `physics/library/`. They are cited by **path**, and by a
  stable heading anchor when one applies:
  `journal/live/audits/2026-07-16-wave2-beta-ga2o3-audit.md`,
  `physics/library/formulas/registry-manifest.csv`. Pages have ids and
  are cited by id — `[deriv-defects] Part G.2`,
  `[timeline] §2026-07-07 (gap audit)` — which is a different rule; the
  two used to be illustrated with each other's examples.
- **Never by line number.** Line references rot on every edit and are
  forbidden in the editable surface (`apparatus.py`; frozen audit
  records under `journal/live/audits/` are exempt as historical text).
- **Count phrasing is canonical** so the count-lint has a contract:
  whole-vocabulary counts are always written `N substantive formulas`,
  `N residual categories`, `N methods`, `N templates`, `N observable
  bundles`, `N cert obligations`. Subset phrasings ("~35 formulas",
  "5–7 bundles") stay un-linted by construction.

## Style

- American English everywhere.
- One **subject** per page, and no subject split across pages. A keystone
  page may own several named topics when they are facets of one machine
  (the substrate, the coupling generator, the three-tier state); that is
  cohesion, not drift. What is forbidden is the same topic owned twice —
  and that is machine-checked.
- **Length.** Canon pages target ~250 lines; a keystone may exceed it
  where splitting would fragment one mechanism. Appendix derivations are
  **exempt** — they are long-form by nature. The earlier flat ~250-line
  rule is retired: the corpus violated it fifteen times, and a rule
  violated fifteen times is not a rule.
- Use ATX headings (`#`, `##`, `###`). Do not use setext (`===` /
  `---` underlines).
- Code fences are bare (no language tag) for the project's
  pseudo-syntax records; tag real code (`python`, `yaml`, `bash`).
- No decorated separator comments anywhere in prose
  (`# ──────`, `# ===`).

## The checker

`python journal/tools/apparatus.py` regenerates `contents.md` and
`index.md` from page frontmatter and restamps `content-hash`.
`--check` verifies without writing, and is the gate. It fails on:

1. missing or unparseable frontmatter, or a missing required field
2. duplicate `id`
3. **duplicate `canonical-for` topic** — the anti-drift invariant
4. asymmetric `depends-on` / `referenced-by`
5. an `[id]` reference in prose resolving to no page
6. a body citing an `[id]` without the matching `depends-on` edge
7. a section coordinate resolving to no heading — including a dated
   anchor whose disambiguating parenthetical matches no entry, and a
   bare date that is ambiguous because several entries share it
8. a line-number citation
9. a registry count claim that disagrees with the CSV, including the
   per-tag diff tallies quoted in prose
10. a stale `content-hash`

The hash is **tool-maintained**: regenerating restamps it, `--check`
verifies it. A stale hash means a page was hand-edited without
regenerating, which is exactly what breaks working-copy staleness
detection — see `journal/instructions.md`, which is a file rather than a
page and so is cited by path, not by id.

Green means the graph is sound. It does **not** mean the physics is
right; that is what `[audit-prompt]` is for, and `[traps]` records what
has gone wrong before.

`seams.py` also lives in `journal/tools/`. It carries the mechanical
sweeps the apparatus does not: row-band claims against the registry CSV,
backticked formula names, `formula = <name>` arguments (declared versus
undeclared), duplicated numeric literals, glossary divergence, retired
formula names (`retired-names.csv`), case-variant near-misses, and `D4`
registry rows whose `source` cell names no relaxation.

**Short symbols are written in backticks.** Diff tags in particular:
`` `D1` `` is a differentiability tag, `D1` unbacked is a wurtzite
deformation potential in `[accuracy-ledger]`. The tag that had no numeric
anchor at all — `DX` — was renamed to `DN` because it collided with the DX
center, a real defect configuration this corpus analyses.

`lint.py` no longer exists. Its unique checks — registry-versus-canon
counts, section-coordinate resolution, the no-line-number-citation rule
— were folded into `apparatus.py`; the rest were either duplicated there
or obsoleted by the move to the book.
